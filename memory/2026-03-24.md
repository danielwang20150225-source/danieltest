## 三、推荐架构：Hub-Agent模式

```
┌─────────────────────────────────────────────────┐
│                 Qclaw Gateway                    │
│                   (单实例)                       │
├─────────────────────────────────────────────────┤
│  ┌──────────────┐   ┌──────────────┐           │
│  │   Main Hub   │──▶│  Sub-Agent 1 │ (简历优化) │
│  │   (主Agent)  │   └──────────────┘           │
│  │              │   ┌──────────────┐           │
│  │  任务分发    │──▶│  Sub-Agent 2 │ (内容创作) │
│  │  结果汇总    │   └──────────────┘           │
│  │              │   ┌──────────────┐           │
│  │              │──▶│  Sub-Agent 3 │ (调研)    │
│  └──────────────┘   └──────────────┘           │
├─────────────────────────────────────────────────┤
│                 飞书 Channel                      │
└─────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 作用 | 实现方式 |
|------|------|----------|
| **Hub Agent** | 主Agent，负责任务分发和结果汇总 | 现有的 main agent |
| **Sub-Agent** | 专业领域执行者 | sessions_spawn |
| **Shared Memory** | 跨Agent记忆共享 | MEMORY.md |
| **Task Router** | 任务路由规则 | Agent指令 |

## 四、配置实现

### 方式1：扩展Agent列表（轻量，推荐）

在 `openclaw.json` 中添加更多Agent：

```json
{
  "agents": {
    "list": [
      { "id": "main" },
      { "id": "investing" },
      { "id": "resume", "name": "简历优化" },
      { "id": "content", "name": "内容创作" },
      { "id": "research", "name": "深度调研" }
    ]
  }
}
```

**优点**：资源极低（1个Gateway进程），每个Agent独立配置
**适用场景**：5-20个Agent

### 方式2：Sub-agent动态Spawn（灵活）

在main Agent中调用 `sessions_spawn` 创建临时子Agent：

```javascript
// 当用户请求涉及"简历"时
await sessions_spawn({
  runtime: "subagent",
  agentId: "resume",  // 调用简历专家
  task: "优化简历：{用户简历内容}",
  mode: "run"  // 完成后自动销毁
})
```

**优点**：按需创建，不占用常驻资源
**适用场景**：复杂任务、多轮对话
