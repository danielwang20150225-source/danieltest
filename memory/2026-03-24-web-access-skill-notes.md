# Web Access Skill 深度学习笔记

> 来源：飞书文档 - WayToAGI
> 日期：2026-03-24

---

## 一、为什么需要 Web Access Skill？

### 现有 Agent 联网方案的痛点

| 方案 | 问题 |
|------|------|
| **Claude Code** | 只提供工具，访问策略全靠模型判断；模型容易钻牛角尖 |
| **OpenClaw** | CDP 模式需要为每个站点重新登录；部分网站组件无法加载；多浏览器会抢控制权 |

### 具体表现
- 模型拿到"调研小红书风评"任务 → 执着用 Search 工具搜非公开内容
- 用 fetch 请求需要登录的网站 → 直接加载失败
- 需要自行安装 Playwright、Chrome Devtool MCP → 多次踩坑
- 并发操作能力差 → 串行处理，token 膨胀

---

## 二、理想方案应该具备的能力

1. **灵活分配工具**：搜索、静态读取、浏览器策略，遇到障碍能自己换工具
2. **复用登录态**：不为每个站点单独维护身份
3. **强泛化能力**：适应不同站点操作和反爬要求
4. **Sub-Agent 分治**：支持高并发跑海量网页，后台执行不抢浏览器
5. **经验沉淀**：积累站点操作经验，下次访问不用从头试错

---

## 三、核心设计理念

```
Skill = 策略哲学 + 最小完备工具集 + 必要的事实说明
```

### 1. 策略哲学：四步循环

```
① 定义成功标准 → 什么算完成了？拿到什么信息？
② 选择起点验证 → 最可能直达的方式作为起点
③ 过程校验 → 每一步的结果都是证据，不只是成功/失败
④ 对照标准停止 → 确认完成后停止，不过度操作
```

**关键点**：
- 不写具体操作路径，只描述"怎么思考"
- 对抗模型的"显式惯性"（模型喜欢用惯性思维一条道走到黑）

### 2. 最小完备工具集

人类上网三种行为 → Agent 需要三种工具：

| 行为 | 工具 | 用途 |
|------|------|------|
| **搜** | Search | 搜索摘要、发现信息来源 |
| **看** | Fetch / curl | 公开页面 AI 提取 / 全文读取 |
| **做** | 浏览器自动化 | 点击、填表、上传等交互操作 |

**额外优化**：
- Jina 作为中间层：把网页正文转成 Markdown，节省 token
- Chrome CDP 原生方案：天然支持单浏览器多 tab 并行，携带登录态

### 3. 必要的事实说明

模型知道很多知识，但并非所有知识都能第一时间调用。

**惰性知识示例**：
- 小红书必须用浏览器访问，Search/Fetch 无效
- 某些网站有反爬机制，需要特殊处理

## 四、子 Agent 分治机制

### 架构
- 主 Agent 分配任务给子 Agent 并行执行
- 所有子 Agent 共享同一个 Chrome / CDP Proxy
- 各自创建后台 tab，通过不同 targetId 操作，互不干扰

### 常见陷阱
主 Agent 给子 Agent 写 prompt 时容易"用词不当"：

❌ 错误示例：
> "在小红书上『搜索』XX 相关信息"

✅ 正确做法：
> "在小红书上调研XX 相关信息"

**原因**：用"搜索"会锚定子 Agent 用 WebSearch，但小红书是反爬平台，正确方法应该用浏览器直接进站内搜索。

## 五、经验沉淀机制

### 工作流程
1. 首次访问站点 → 用通用思路试错
2. 操作完成后 → 自动把站点访问策略沉淀到文件
3. 下次访问 → 复用先验知识，跳过试错环节

### 记录内容
- 平台特征
- 有效的 URL 模式
- 已知的陷阱

### 效率对比
| 模式 | 步骤数 | 耗时 |
|------|--------|------|
| 无经验 | 15+ | 长 |
| 有经验 | 3-5 | 短（节省 ~90%）|

### 经验文件设计
- 标注发现日期（作为"可能有效的提示"而非"保证正确"）
- 网站改版失败 → 自动回退通用模式并更新经验

---

## 六、架构思考：在线经验共享池

### 构想
- 所有人共享站点操作经验
- 每次操作都在让池子变得更好
- 按需拉取，不用时不占用上下文

### 阻碍：法律风险
- 通用联网指导 vs 针对特定站点的结构分析经验
- Agent 代查代发是否合规？

## 七、安装配置

### 安装命令
```
帮我安装 web-access skill，仓库地址是 https://github.com/eze-is/web-access
```

### 前置要求
- Chrome 浏览器 + 更新到最新
- 打开 `chrome://inspect/#remote-debugging`，勾选 `Allow remote debugging`

### 使用方式
- 输入"遵循 web-access skill"手动要求
- 或直接输入联网任务

---

## 八、核心收获

1. **Skill 设计公式**：策略哲学 + 最小完备工具 + 必要事实说明
2. **四步循环**：定义成功 → 起点验证 → 过程校验 → 对照停止
3. **工具三要素**：搜、看、做
4. **子 Agent 分治**：写目标不写步骤，注意用词陷阱
5. **经验沉淀**：让 Agent 越用越聪明
