## 一、项目概述

**项目名称**：魔法简历
**项目类型**：AI + 求职服务
**核心定位**：基于 AI 的简历优化工具 → 扩展到求职服务生态

**商业模式**：
- 初期（A）：AI 简历匹配分析工具（免费/低价获客）
- 中期（B）：人工简历优化服务（付费变现）
- 长期（C）：求职培训、社群等生态

**目标用户**：社招/跳槽求职者（产品经理/互联网行业）

**核心价值**："让面试官能通过简历认识到我，发现我的亮点"

## 三、产品定位

### MVP 切入点
**先解决一个问题**：简历投了很多份，但没有面试邀约

### 核心功能
1. **简历上传** - 支持 PDF/Word/图片格式
2. **岗位 JD 输入** - 粘贴目标岗位描述
3. **AI 匹配分析** - 匹配度评分、关键词缺失提醒
4. **优化后的简历输出** - 带颜色标注
5. **优化说明** - 每处修改的原因
6. **灵活调整** - 自然语言进一步优化

### 输出格式（颜色标注）
- 🔴 红色：新增内容
- 🟠 橙色：修改内容
- ⚫ 灰色：删除内容

### 解决的5个核心问题
1. **差距认知** → 匹配项/差距项清晰展示
2. **优化透明** → 每处修改都有原因解释
3. **灵活调整** → 支持自然语言微调
4. **差异化** → 突出个人亮点
5. **ATS适配** → 通过机器初筛

### 交互设计
- 优化后的简历界面
- 用户可选择具体段落/句子
- 对话框输入调整需求
- 参考"邮件AI润色"交互

## 八、产品场景定义（2026-03-12 总结）

### 场景1：简历优化服务
**输入**：JD + 简历 → **输出**：个性化深度匹配JD的专属简历
- 核心：基于JD要求，优化简历内容
- 交互：优化后支持2次编辑对话

### 场景2：能力提升服务
**问题**：优化中发现用户与JD存在真实差距
**解决**：提供知识/方法论/作品/行动方案
- **知识**：补充缺失领域的基础认知
- **方法论**：行业/岗位核心方法
- **作品**：引导产出面试作品（如产品分析报告）
- **行动方案**：快速弥补差距的具体路径

### 两个场景的关系
- 场景1是**表面需求**（用户想要一份好简历）
- 场景2是**深层需求**（用户想要真正匹配岗位的能力）
- 产品价值：场景1获客 → 场景2转化付费
