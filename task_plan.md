## 技术选型方案

### 方案A：Next.js全栈
| 优点 | 缺点 |
|------|------|
| 前后端一体化，开发快 | 部署灵活性较低 |
| Vercel部署简单 | 冷启动慢 |
| AI API对接方便 | 定制化有限 |

### 方案B：React + Node.js
| 优点 | 缺点 |
|------|------|
| 前后端分离，灵活 | 开发复杂度高 |
| 可部署到任意服务器 | 需要更多配置 |
| 适合大规模应用 | 人力需求高 |

### 推荐：方案A (Next.js)
理由：
1. MVP阶段快速上线
2. 前后端一体，减少协调成本
3. Vercel免费额度够用

---

| 层 | 技术 | 理由 |

## 技术难点

1. **PDF/DOCX解析** 
   - Node环境兼容性
   - OCR识别准确率
   - 解决：pdf-parse@1.1.0 + Tesseract.js

2. **AI输出稳定性**
   - Prompt调优成本
   - 输出格式一致性
   - 解决：严格Prompt + 输出校验

3. **实时交互性能**
   - Hover/气泡响应
   - 大量文本渲染
   - 解决：虚拟列表 + 防抖

4. **大规模并发**
   - AI API限流
   - 成本控制
   - 解决：队列机制 + 缓存

5. **数据安全**
   - 简历隐私保护
   - 传输加密
   - 解决：HTTPS + 加密存储
