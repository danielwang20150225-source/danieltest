## 一、产品概述

### 1.1 产品定位
AI驱动的简历优化工具，帮助求职者基于目标岗位JD优化简历，突出核心竞争力。

### 1.2 核心价值
"让面试官能通过简历认识到你，发现你的亮点"

### 1.3 目标用户
- 社招/跳槽求职者
- 产品经理/互联网行业
- 转行者

## 三、用户流程

### 3.1 主流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  上传简历   │ ──→ │  输入目标JD  │ ──→ │  AI优化分析 │
└─────────────┘     └─────────────┘     └─────────────┘
                                                ↓
                    ┌─────────────┐     ┌─────────────┐
                    │  灵活调整   │ ←── │  输出优化简历│
                    └─────────────┘     └─────────────┘
                           ↓
                    ┌─────────────┐
                    │  场景2引导  │ ← 可选
                    └─────────────┘
```

### 3.2 场景2流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  选择类型   │ ──→ │  输入领域   │ ──→ │  输出内容   │
│(知识/方法/ │     │(可穿戴/A/B  │     │ (知识/方法/ │
│ 作品/行动) │     │  实验等)    │     │  作品/计划) │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 五、数据模型

### 5.1 用户输入
```typescript
interface ResumeInput {
  content: string;      // 简历文本
  format: 'pdf' | 'docx' | 'image' | 'text';
  filename?: string;
}

interface JDInput {
  content: string;      // JD文本
  title?: string;       // 岗位名称
  company?: string;     // 公司名称
}
```

### 5.2 AI输出
```typescript
interface OptimizeResult {
  matchScore: number;    // 匹配度 0-100
  matchedItems: string[];
  missingItems: string[];
  migratableItems: string[];
  optimizedResume: string;
  changes: Change[];
  suggestions: Suggestion[];
}

interface Change {
  type: 'add' | 'modify' | 'delete';
  content: string;
  reason: string;
  jdMatch: string;
}
```

## 七、非功能性需求

### 7.1 性能
- 简历解析：< 3秒
- AI优化：< 30秒
- 页面加载：< 2秒

### 7.2 准确率
- 文件解析准确率：> 95%
- JD识别准确率：> 90%

### 7.3 安全
- 用户数据加密存储
- 简历内容不共享
- API调用日志脱敏

## 八、竞品分析

| 产品 | 优势 | 劣势 |
|------|------|------|
| 超级简历 | 模板丰富 | AI能力弱 |
| Resume.io | 英文优化强 | 仅有格式 |
| 竞品AI | 通用优化 | 无行业深度 |

**差异化机会**：
- 垂直行业（产品经理）
- 深度内容优化
- 能力提升服务

## 九、风险与对策

| 风险 | 对策 |
|------|------|
| 简历隐私泄露 | 加密存储，用完即删 |
| AI幻觉/虚构 | 严格Prompt约束 |
| 用户期望过高 | 明确"优化"vs"虚构"边界 |
