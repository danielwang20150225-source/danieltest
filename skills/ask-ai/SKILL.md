## 执行流程

### 阶段1：打开浏览器并导航

**步骤1.1：打开新窗口或新标签**
```
使用 browser 工具:
- action: "open" 或 "start"
- targetUrl: "https://www.doubao.com/chat/"
- profile: "chrome"  # 使用默认Chrome配置
```

**步骤1.2：等待页面加载**
```
等待3-5秒，让页面完全加载
检查页面是否出现输入框元素
```

### 阶段2：定位并操作输入框

**步骤2.1：获取页面快照**
```
使用 browser 工具:
- action: "snapshot"
获取页面元素，找到输入框的引用（如 @e1）
```

**步骤2.2：填写问题**
```
使用 browser 工具:
- action: "act"
- ref: "@e1"  # 输入框引用
- text: "你好，我有一个技术问题想请教：\n\n【问题】\n{用户的问题}\n\n请给出详细解答。"
```

**步骤2.3：找到发送按钮并点击**
```
使用 browser 工具:
- action: "snapshot"  # 重新获取页面元素
找到发送按钮的引用（如 @e2）
- action: "act"
- ref: "@e2"  # 发送按钮引用
- action: "click"
```

### 阶段3：等待并读取响应

**步骤3.1：轮询等待AI响应**
```
每20秒执行一次 browser snapshot:
- 检查是否有新的AI回复内容
- 验证是否出现loading状态消失

最多轮询7次（140秒）
```

**步骤3.2：读取响应内容**
```
从页面快照中提取AI的回复文本
记录关键信息
```

### 阶段4：多轮追问（如需要）

**步骤4.1：发送追问**
```
继续在输入框填写追问内容
重复2.2-2.3步骤
```

**步骤4.2：结束对话**
```
获取完整对话内容
关闭浏览器或保持页面
```

## 强制执行检查清单

### 检查点1：发送完成标识
- 必须在点击发送按钮后输出：【发送完成】
- 未看到发送按钮？尝试按Enter键提交

### 检查点2：等待响应
- 豆包通常5-30秒内回复
- 超过1分钟无响应？刷新页面重试

### 检查点3：验证码/登录检测
- 发现验证码弹窗 → 立即停止，告知用户帮忙
- 发现需要登录 → 尝试自动登录或请求用户协助

---

## 错误处理

| 错误 | 处理方式 |
|------|----------|
| 页面加载超时 | 等待10秒后重试，共3次 |
| 找不到输入框 | 刷新页面，重新获取快照 |
| 发送失败 | 重试2次，仍失败切换下一个AI |
| AI无响应 | 切换到下一个AI（千问/Kimi/DeepSeek）|

## 提问模板

```
你好，我有一个技术问题想请教：

【背景】
- 用户想实现：{具体需求}
- 我是AI助手

【问题】
- {问题描述}

【请帮我】
1. 分析根因
2. 给出可行方案
3. 指出可能的坑
```

---

## 注意事项

1. **不要白嫖**：使用用户的AI账号对话
2. **有记录**：把AI的回复保存到memory
3. **闭环**：拿到方案后要执行，不能只给答案
4. **真实优先**：示例场景要用真实案例，不要编
5. **心跳轮询**：等待时必须轮询，不要傻等
6. **验证码处理**：遇到验证码立即停止并告知用户
7. **告知用户**：所有AI都失败后，明确告知用户当前状态和建议

## 示例场景

### 场景1：问豆包技术问题

```
用户："帮我问下豆包，如何让OpenClaw拉起独立浏览器窗口？"

执行步骤：
1. browser.open -> doubao.com/chat/
2. 等待页面加载
3. browser.snapshot -> 找到输入框 @e1
4. browser.fill(@e1, "你好，我有一个技术问题...")
5. browser.snapshot -> 找到发送按钮 @e2  
6. browser.click(@e2)
7. 等待AI响应...
8. 读取响应内容
9. 返回给用户
```

### 场景2：豆包无响应，切换千问

```
1. 问豆包 -> 30秒无响应
2. 切换到千问: browser.open -> qianwen.aliyun.com
3. 重复上述流程
4. 仍无响应？切换Kimi...
```
